# Checklist de SEO Técnico y Filosofía de Contenido

Este documento sirve como una guía de referencia para asegurar que todo el contenido nuevo y las páginas del sitio web estén optimizadas para los motores de búsqueda y, lo más importante, para los usuarios.

---

## 1. Filosofía de Contenido (Inspirada en Nandark)

El SEO moderno, especialmente en la era de la IA, ya no se trata solo de repetir palabras clave. Se trata de **resolver problemas** y **generar confianza**. Antes de escribir cualquier pieza de contenido, pregúntate:

*   **¿Qué problema real de mi cliente estoy resolviendo con este contenido?**
    *   **Enfoque incorrecto:** "Hablemos de las características de la vitamina C."
    *   **Enfoque correcto:** "¿Tienes manchas oscuras que no desaparecen? La vitamina C podría ser tu solución. Así es como funciona..."

*   **¿Estoy demostrando autoridad y confianza?**
    *   El contenido debe ser preciso, bien investigado y transparente. Explica el "porqué" detrás de tus recomendaciones.
    *   Sé claro sobre tu experiencia y credenciales.

*   **¿Estoy facilitando que tanto los usuarios como los motores de búsqueda entiendan el contenido?**
    *   Usa un lenguaje claro y accesible. Estructura tu contenido de manera lógica.

En resumen, el objetivo es convertirte en la **respuesta más confiable y útil** para las preguntas y problemas de tu audiencia. Si lo haces, los motores de búsqueda te recompensarán.

## 2. Checklist de SEO Técnico y de Contenido

Utiliza esta lista para revisar cada página y publicación de blog antes y después de publicarla.

### Contenido y Palabras Clave

- [x] **Investigación de Palabra Clave Principal:** ¿La página se enfoca en una palabra clave principal (ej. "rutina piel seca")?
- [x] **Título Principal (H1):** ¿La página tiene un único H1 que incluye la palabra clave principal?
- [x] **Subtítulos (H2, H3):** ¿El contenido está estructurado con subtítulos que usan variaciones de la palabra clave?
- [x] **Densidad de Palabra Clave:** ¿La palabra clave principal aparece de forma natural en la introducción, en el cuerpo del texto y en la conclusión?
- [x] **Contenido de Calidad:** ¿El contenido es original, útil, bien investigado y resuelve la intención de búsqueda del usuario?
- [ ] **Enlaces Internos:** ¿El post enlaza a otras páginas o posts relevantes dentro de tu propio sitio web?
- [ ] **Enlaces Externos:** ¿El post enlaza a fuentes externas de alta autoridad (si es aplicable)?

### Metadatos para SEO

- [x] **Meta Título (`<title>`):** ¿Tiene entre 50 y 60 caracteres e incluye la palabra clave principal al principio?
- [x] **Meta Descripción (`<meta name="description">`):** ¿Tiene entre 150 y 160 caracteres, es atractiva e incluye la palabra clave principal?

### Optimización para Redes Sociales

- [x] **Open Graph - Título (`og:title`):** ¿Está definido y es atractivo para compartir en redes como Facebook?
- [x] **Open Graph - Descripción (`og:description`):** ¿Está definida y resume el contenido?
- [x] **Open Graph - Imagen (`og:image`):** ¿Se ha especificado una imagen de alta calidad (1200x630px)?
- [x] **Twitter Card - Título (`twitter:title`):** ¿Está definido?
- [x] **Twitter Card - Descripción (`twitter:description`):** ¿Está definida?
- [x] **Twitter Card - Imagen (`twitter:image`):** ¿Está especificada?

### Datos Estructurados (Schema.org)

- [x] **JSON-LD:** ¿La página incluye un script de JSON-LD que describe el contenido? (Implementado para posts de blog. Pendiente para otras páginas.)
    -   Para posts de blog: Usar el schema `BlogPosting`.
    -   Para páginas de servicios: Usar el schema `Service`.
    -   Para la página de contacto: Usar el schema `ContactPage`.
    -   Para la página "Sobre Mí": Usar el schema `AboutPage`.

### Aspectos Técnicos

- [x] **URL Amigable:** ¿La URL es corta, descriptiva e incluye la palabra clave principal (ej. `/blog/rutina-piel-seca`)?
- [ ] **Imágenes Optimizadas:**
    -   ¿Las imágenes están comprimidas para una carga rápida?
    -   ¿Los nombres de archivo son descriptivos (ej. `rutina-skincare-piel-seca.jpg`)?
    -   ¿Todas las imágenes tienen texto alternativo (`alt text`) descriptivo?
- [ ] **Diseño Responsivo:** ¿La página se ve y funciona bien en dispositivos móviles?
- [ ] **Velocidad de Carga:** ¿La página carga en menos de 3 segundos? (Puedes usar herramientas como Google PageSpeed Insights para medirlo).
- [x] **URL Canónica (`<link rel="canonical">`):** ¿La página tiene una URL canónica para evitar contenido duplicado?
